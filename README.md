# 研面：电子信息保研模拟面试 Agent

## 项目简介

研面是一个面向电子信息保研准备的本地单用户网页应用。它使用阿里云百炼（Qwen）完成材料智能提取、实时模拟面试和证据化复盘；浏览器通过本地服务连接模型，长期 API Key 不会进入前端。

应用把个人材料整理成可核验的画像事实和详细经历卡，再由主考官、专业老师、科研老师和英语老师进行 10、20 或 30 分钟模拟面试。面试结束后，四位评审根据转写生成六维评分、优先改进项、示范回答和七天训练计划。

## V1.1 面试覆盖规则

当前版本为 **V1.1**。专业基础、科研项目和英语交流三个核心模块按面试时长设置新主题覆盖目标：

| 面试时长 | 专业基础 | 科研项目 | 英语交流 |
| --- | ---: | ---: | ---: |
| 10 分钟 | 1 个主题 | 1 个主题 | 1 个主题 |
| 20 分钟 | 2 个主题 | 2 个主题 | 2 个主题 |
| 30 分钟 | 3 个主题 | 3 个主题 | 3 个主题 |

英语模块用于日常与综合交流能力考察，并遵守以下范围：

- 允许：个人与校园生活、性格品质、学习方法、经历感受、团队协作、报考动机、读研规划、宽泛研究兴趣、压力应对和日常沟通。
- 禁止：专业课程或专业技能、论文或发表、具体项目/竞赛/实验、研究方法、算法、技术细节、电路和协议等专业技术问答。

主题数是模块内的新主题覆盖目标，不会为了凑数打断候选人较长但相关的回答，也不会覆盖最后一分钟的强制收尾规则；收尾优先于尚未完成的主题目标。

实时语音与文字模式共享同一份覆盖状态。模式切换不会重置已完成主题、已用英语题目或当前追问进度。

## 核心能力

- 本地解析 PDF、DOCX、JPG/JPEG、PNG、TXT 和 Markdown；单个文件最大 20 MB，并可分类为个人材料、目标院校或专业参考资料。
- 从材料提取带来源、页码和置信度的事实；低置信度或冲突信息由用户编辑、确认。
- 为科研、项目和竞赛经历生成包含背景、个人职责、方法、成果及奖项/角色的详细经历卡。
- 以主考官、专业老师、科研老师和英语老师四个角色开展实时面试，支持单主题最多三层追问、断线暂停和最后一分钟收尾。
- 麦克风不可用时降级为文字回答；不在本地保存原始音频。
- 保存面试转写与复盘，并按相同方向、时长和压力等级展示长期趋势。

## 快速开始

以下命令以 Windows PowerShell 为例。项目当前开发环境使用 Node.js 24 和 npm 11。

1. 进入项目目录并安装依赖：

   ```powershell
   npm install
   ```

2. 复制环境变量模板：

   ```powershell
   Copy-Item .env.example .env.local
   ```

3. 在仅保存在本机的 `.env.local` 中填写 `ALIYUN_API_KEY` 或 `DASHSCOPE_API_KEY`。两者同时存在时 `ALIYUN_API_KEY` 优先。不要填写示例值，不要把真实 Key 提交到 Git 或发送到聊天中。其余 `QWEN_*` 配置通常可先沿用模板默认值。

4. 启动本地服务：

   ```powershell
   npm run dev
   ```

5. 浏览器打开 [http://127.0.0.1:3000](http://127.0.0.1:3000)。

修改 `.env.local` 或系统环境变量后，需要停止服务并重新执行 `npm run dev` 才能确保新值生效。真实百炼请求按账号计费，请先确认 Key、地域和计费状态。

## 推荐使用流程

1. 打开“材料与个人画像”，点击“选择本地文件”，选择材料类别，再点击“本地解析并建立索引”。先等待页面提示解析完成；若显示“智能解析待重试”，按下文说明重试。
2. 检查“个人画像事实”的值、来源和置信度。修正不准确内容后逐条“确认”；未确认事实不会被当作确定事实使用。
3. 对以前已完成智能解析的个人材料，点击材料旁的“重新提取”，补建或刷新详细经历卡；处于待重试状态时，同一按钮显示“重试”。
4. 展开每张草稿卡，核对经历类型、标题、背景、个人职责、方法与过程、量化成果和奖项/角色。使用“保存修改”保留草稿；确认全部字段可信后点击“确认整段经历”。只有整卡确认的经历会作为面试中的已确认经历。
5. 上传错误时，点击材料旁的“删除”并确认。此操作会永久删除该材料及其画像事实和详细经历；需要保留的内容请先核对。
6. 回到首页，选择 10、20 或 30 分钟，填写考察重点和压力等级，并按需选择引用材料，然后点击“开始模拟面试”。允许麦克风权限；无法使用麦克风时改用面试页底部的文字输入框。
7. 结束面试后等待四位评审生成报告。阅读六维评分、优先改进、示范回答和七天训练计划；随后进入“复盘记录”查看历史报告与同配置长期趋势。若报告标为部分完成，不生成正式总分，可点击“重试未完成评审”。

## 详细经历卡与项目面试

智能提取不依赖项目名称白名单，而是从材料语义和同页逐字证据生成经历卡。模型没有证据的详情会留空；页面的“待补”提示表示仍需人工补全。保存草稿只保存编辑内容，“确认整段经历”才把整张卡标为已确认；已确认卡仍可选择“重新编辑”。

创建面试时，科研老师只从本轮所选材料的已确认经历中选择首问对象。优先级依次为：与“考察重点”的相关性、详情完整度、成果中是否包含量化数字、创建时间新旧，最后以稳定 ID 排序。科研模块第一问会点名排在最前的经历，并询问候选人的个人职责；没有已确认经历时不会虚构这一首问依据。

## 测试与真实验收

依次运行自动检查：

```powershell
npm test
npm run lint
npm run build
npm run test:e2e
```

`npm test` 运行 Vitest，`npm run lint` 执行 TypeScript 类型检查，`npm run build` 验证生产构建，`npm run test:e2e` 运行 Playwright 端到端测试。Playwright 默认使用本机 Chrome。真实百炼冒烟测试会产生费用，不包含在这些默认命令中。

材料会先完成本地解析，再分块尝试智能提取。状态为“智能解析待重试”或 `basic_only`，表示至少一个智能分块失败；成功分块恢复出的事实或经历仍可能已经保存，但结果可能不完整。检查已有内容后，点击该个人材料的“重试”，不要把部分状态理解为完整成功。

真实百炼验收（经授权材料）：一份简历在一个成功分块中完成解析；恢复了 Super-LoRa、嵌入式 FPGA 竞赛和电子设计 G 题的详细经历卡；一次生产文本面试请求点名 Super-LoRa，并询问个人职责。

## 隐私与本地数据

“本地保存”不等于“所有内容永不离开电脑”。边界如下：

- 原始 PDF、DOCX、图片或文本文件不会上传到百炼，保存在 `%LOCALAPPDATA%\BaoyanInterviewAgent\uploads\`；SQLite 数据库默认位于 `%LOCALAPPDATA%\BaoyanInterviewAgent\baoyan.db`。设置 `BAOYAN_DATA_DIR` 可覆盖数据根目录。
- 智能材料提取会从本地解析文本中选择与经历有关的候选段落，移除已识别的姓名、电话、邮箱、地址和社交账号等联系方式，再按大小分块发送到百炼。自动脱敏不能替代人工审查，不应上传无关敏感材料。
- 进行面试和复盘时，考察重点、所选材料的相关文本片段、已确认事实与经历卡、文字回答和面试转写可发送到百炼作为上下文；实时语音模式也会把音频流发送给百炼处理。
- 原始材料、提取结果、经历卡、面试转写和复盘报告均在本地数据库或数据目录中持久化；原始音频不在本地落盘。删除材料会连同其派生画像事实和经历卡删除。
- API Key 只由本地服务读取，不进入浏览器前端。`.env.local` 和本地数据目录不得提交到版本库，也不应包含在问题截图或日志中。

## 常见问题

**端口 3000 已被占用**

先停止占用该端口的进程，或在当前 PowerShell 会话指定其他端口后启动：

```powershell
$env:PORT=3001
npm run dev
```

随后打开 `http://127.0.0.1:3001`。

**提示缺少 API Key 或环境变量修改后仍报错**

确认 `.env.local` 中至少填写了 `ALIYUN_API_KEY` 或 `DASHSCOPE_API_KEY`，没有引号、尾部空格或示例占位值；两者同时填写时检查优先使用的 `ALIYUN_API_KEY`。停止并重新启动本地服务。若使用系统环境变量，也要重新打开 PowerShell 后再启动。

**材料显示“智能解析待重试”**

本地文本和索引已经建立，但一个或多个智能分块失败，已有智能结果可能不完整。检查已生成内容，再点击个人材料旁的“重试”；同时检查网络、Key、地域、模型配置和百炼余额。重复上传同一文件会被判定为重复材料，不用于替代重试。

**麦克风不可用或没有声音**

在浏览器站点设置中允许 `127.0.0.1` 使用麦克风，并确认 Windows 输入设备和权限。仍不可用时使用页面底部文字输入；应用不会在本地保存原始音频。

**复盘只完成一部分**

部分评审完成时未知维度不会按 0 分计算，也不会生成正式总分。转写和已完成结果仍保存在本地，可在报告或“复盘记录”中重试未完成评审。

## 当前版本边界

- 面向本机单用户使用，不提供多账号、云同步或跨设备共享。
- 不执行联网院校检索，也不保证材料事实自动正确；低置信度、冲突事实和经历卡必须人工核对后确认。
- 模拟评分与训练建议用于练习，不代表院校评价、录取概率或正式结论。
- 智能提取、实时面试和复盘依赖阿里云百炼的网络、模型可用性、配额与计费状态；离线时只能使用已经落在本地的数据，不能完成这些模型调用。
