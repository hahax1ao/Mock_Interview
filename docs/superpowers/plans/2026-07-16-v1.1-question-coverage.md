# V1.1 English Boundaries and Question Coverage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make 10/20/30-minute interviews reliably cover technical, research/project, and English topics while restricting English questions to non-technical daily communication, motivation, personal qualities, and future planning.

**Architecture:** Add a pure question-coverage domain module that owns duration targets, topic rotation, English question selection, follow-up depth, and control-instruction generation. Persist each issued topic decision as a validated interview event so realtime reconnects and text fallback reconstruct the same coverage state. Realtime and text paths call the same scheduler; the UI only displays its duration targets.

**Tech Stack:** Next.js, TypeScript, React, Zod, SQLite/Drizzle ORM, Vitest, Playwright, Alibaba Cloud Bailian Realtime and OpenAI-compatible text API.

## Global Constraints

- English must exclude major/course, professional-skill, paper, research-project, competition-project, and technical-detail questions.
- The formal English question bank must be version-controlled under `src/`; runtime code must not read `test_thing/english_interview_questions.txt`.
- Minimum new-topic targets are exactly 1/1/1 for 10 minutes, 2/2/2 for 20 minutes, and 3/3/3 for 30 minutes across technical/research/English.
- A long candidate answer must not be interrupted merely to reach the target.
- The final minute always belongs to the chair and cannot be extended to complete coverage.
- Each topic allows at most three follow-up questions.
- Realtime voice and text fallback must share the same scheduler and persisted coverage events.
- Research questions use confirmed detailed experiences when available and must not invent a project when none exists.
- Existing V1 interviews and review records remain readable.
- Default automated tests use mocks; real Bailian calls remain manual and billable.

```ts
import {
  selectEnglishQuestion,
  type EnglishQuestionCategory,
} from "./english-question-bank";

export type CoreInterviewRole = "technical" | "research" | "english";
export type QuestionControl = {
  role: CoreInterviewRole | "chair";
  kind: "new_topic" | "follow_up" | "closing";
  topicId: string;
  topicCategory: string;
  questionId?: string;
  questionText?: string;
  followUpDepth: number;
  issuedAtMs: number;
};

export type CoverageState = {
  topicCounts: Record<CoreInterviewRole, number>;
  usedTopicIds: Record<CoreInterviewRole, string[]>;
  usedEnglishQuestionIds: string[];
  usedEnglishCategories: EnglishQuestionCategory[];
  current?: QuestionControl;
};

const TARGETS = { 10: 1, 20: 2, 30: 3 } as const;
const TECHNICAL_TOPICS = ["signals", "communications", "digital", "analog", "circuits", "probability"] as const;
const RESEARCH_TOPICS = ["responsibility", "method", "validation", "limitations", "teamwork", "planning"] as const;

export function topicTargetsForDuration(duration: 10 | 20 | 30) {
  const target = TARGETS[duration];
  return { technical: target, research: target, english: target };
}

export function rebuildCoverageState(controls: QuestionControl[]): CoverageState {
  const newTopics = controls.filter((control) => control.kind === "new_topic");
  const idsFor = (role: CoreInterviewRole) =>
    newTopics.filter((control) => control.role === role).map((control) => control.topicId);
  const english = newTopics.filter((control) => control.role === "english");
  return {
    topicCounts: {
      technical: idsFor("technical").length,
      research: idsFor("research").length,
      english: idsFor("english").length,
    },
    usedTopicIds: {
      technical: idsFor("technical"),
      research: idsFor("research"),
      english: idsFor("english"),
    },
    usedEnglishQuestionIds: english.flatMap((control) => control.questionId ? [control.questionId] : []),
    usedEnglishCategories: english.map((control) => control.topicCategory as EnglishQuestionCategory),
    current: controls.at(-1),
  };
}
```

Implement `decideNextQuestion` so that:

1. `elapsedMs >= duration * 60_000 - 60_000` returns `chair/closing`.
2. A role below its target starts a new topic before any second follow-up; this guarantees coverage without interrupting the current answer.
3. A role at its target can follow up until depth 3.
4. Depth 3 always starts another unused topic.
5. English new topics use `selectEnglishQuestion`.
6. Technical/research topics choose the first unused ID from their deterministic pools.
7. `issuedAtMs` equals the provided `elapsedMs`.
