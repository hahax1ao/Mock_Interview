# 项目 README 重写设计

## 目标

将 README 重写为面向项目使用者的中文入口文档。读者应能快速理解项目用途、完成本地启动、上传并确认材料、开始模拟面试，并理解真实百炼调用与本地数据的隐私边界。

## 读者

- 首次运行项目的本人或开发者
- 需要确认材料解析、详细经历卡和模拟面试工作流的测试者
- 需要检查本地存储、百炼调用和 API Key 安全边界的维护者

## 文档结构

1. 项目简介：说明这是本地单用户的电子信息保研模拟面试 Agent，模型服务使用阿里云百炼。
2. 核心能力：概述材料解析、详细经历卡、四角色面试、证据化复盘和长期训练。
3. 快速开始：列出环境要求、依赖安装、环境变量、启动地址。
4. 推荐使用流程：上传材料、检查并确认画像事实、确认详细经历卡、创建面试、查看复盘。
5. 详细经历与面试规则：解释通用经历提取、草稿编辑、整卡确认和科研老师优先提问。
6. 验证与真实验收：列出自动测试命令，并记录真实百炼已成功识别三类目标经历以及科研首问集成结果。
7. 隐私与本地数据：明确原文件不上传百炼、文本按需脱敏分块发送、原始音频不落盘、API Key 不进入前端。
8. 常见问题：覆盖端口占用、环境变量重启、解析待重试、麦克风降级和真实调用费用。

## 写作约束

- 使用简洁、可执行的中文，不复述内部开发过程。
- 命令以 PowerShell 为默认环境。
- 不写入真实 API Key、完整简历内容或个人联系方式。
- 不承诺录取概率、联网检索或当前版本未实现的能力。
- README 采用 UTF-8，修正当前显示异常的中文内容。

## 验收标准

- 新读者只看 README 即可启动应用并完成一次材料到面试的流程。
- 详细经历卡的草稿、确认、删除、重提取和面试优先级说明准确。
- 自动测试命令、真实验收结论和隐私边界与当前实现一致。
- 文档不存在占位符、相互矛盾的说明或未实现功能。
