# 材料管理与混合解析设计

## 目标

改进本地材料库的可维护性与画像事实的准确性：阻止完全相同的文件被重复上传，允许用户安全删除误传材料，并采用“本地规则 + 百炼结构化抽取 + 原文证据校验”的混合方案解析复杂简历。

本次改动仅涉及材料上传、解析、事实展示与删除，不改变已有面试、转写和复盘流程。

## 已确认行为

- 以文件内容哈希判断重复。内容完全相同的文件只保留一份；文件名相同但内容不同可以上传。
- 删除前必须二次确认。
- 删除材料时永久删除原文件、检索片段及该材料产生的全部画像事实。
- 已有面试与复盘记录保留，但新场次不能引用已删除材料。
- 允许把本地提取出的材料文本发送给阿里云百炼，以提高复杂内容的结构化抽取质量；原始文件不发送给百炼。
- 百炼不可用时材料仍保存在本地并完成基础解析，界面明确显示智能解析未完成，且允许稍后重试。

## 上传与去重

服务端收到文件后，先对原始字节计算 SHA-256 哈希，再进行文件写入或模型调用。

材料表新增可空的 `content_hash` 字段，以兼容现有数据库。首次处理新上传时，系统会为仍缺少哈希的现有材料读取本地文件并补算哈希。若新文件的哈希已存在，接口返回冲突状态和已有材料的信息，不写入文件、不重复解析，也不调用百炼。

只有通过去重检查后，系统才创建材料目录并继续解析。材料数据库记录保存解析状态，用于区分“完整解析”和“仅完成基础解析”。

## 混合解析

### 本地基础解析

PDF、DOCX、图片、TXT 和 Markdown 继续在本机完成文本提取或 OCR，并保留页码。排名、GPA、均分、四六级、课程等格式明确、可直接验证的字段由本地规则提取。

本地规则按行和字段边界匹配，不再跨越到下一个标题。值为标题、空值、姓名或明显不属于该字段的内容时直接拒绝，从根源上避免把“个人技能：”“其它经历：”或“姓名 沈笑”误识别为科研、技能或竞赛内容。

### 百炼智能解析

项目、科研、竞赛、技能和荣誉等复杂字段交给百炼文本模型进行结构化抽取。模型名称集中配置为 `materialExtraction`，默认复用经济型文本模型，便于以后单独升级。

每条模型结果必须包含：

- 字段类型；
- 结构化值；
- 原文证据片段；
- 页码；
- 模型置信度。

响应使用 Zod 校验。服务端还会验证证据片段能够在对应页的本地文本中找到；缺少证据、字段类型非法、值只是标题或内容为空的结果全部丢弃。模型不能凭常识补充材料中不存在的信息。

### 合并与冲突

本地结果和百炼结果按照“标准化字段 + 标准化值”去重。相同事实只保存一次；字段相同但值不同的结果作为冲突事实保留，均保持未确认状态并交给用户判断。

现有人工确认流程保持不变。每条事实继续记录材料来源、置信度和确认状态。

## 删除流程

新增 `DELETE /api/materials/:id`。

删除服务首先确认材料存在，并校验文件路径位于应用的本地上传目录内。为避免数据库和文件只删除一半，材料目录先原子移动到应用内的临时隔离目录，然后在数据库事务中删除材料记录。外键级联同步删除检索片段与画像事实。

数据库删除失败时，隔离目录恢复到原位置并返回错误。数据库删除成功后，彻底删除隔离目录。若最后的物理清理失败，接口记录错误但材料在应用中已不可见，后续启动时清理遗留隔离目录。

历史面试的材料 ID 快照和复盘报告不删除。创建新场次时已有的存在性校验继续阻止引用已删除材料。

## 界面行为

- 每个材料行增加明确的“删除”操作。
- 点击后显示包含文件名的二次确认。
- 删除进行中禁用该按钮，防止重复请求。
- 成功后立即从材料列表、画像事实列表、面试材料选择和当前选择状态中移除对应数据。
- 完全重复上传时显示已有材料名称和上传时间，而不是通用解析错误。
- 上传结果区分“智能解析完成”和“基础解析完成，智能解析待重试”。
- 画像事实按材料来源保持可辨识，避免多份不同材料产生相似字段时难以追踪。

## 接口与结构

### `POST /api/materials`

在原有响应上增加：

- `parseStatus: complete | basic_only`
- `smartFacts: number`
- 重复文件时返回 HTTP 409，并提供 `duplicateMaterial` 摘要。

### `DELETE /api/materials/:id`

- 成功返回被删除材料的 ID。
- 材料不存在返回 HTTP 404。
- 非法路径或删除过程失败返回可读错误，不进行越界文件操作。

### `POST /api/materials/:id/retry`

只对现有材料重新运行百炼智能抽取，不重新上传文件，也不重复保存已经存在的事实。

## 错误处理与隐私

- 原始材料、全文索引、哈希和数据库只保存在本机。
- 百炼只接收本地解析后的文本及任务说明，不接收原文件路径或原始二进制文件。
- 百炼密钥仅从服务端环境变量读取。
- 百炼超时、限流或结构化输出非法时降级为基础解析，不回滚已成功保存的材料。
- 日志不记录材料全文、API Key 或个人敏感字段。

## 测试与验收

- 单元测试验证内容哈希、现有材料哈希补全、同内容拦截和同名不同内容放行。
- 解析测试覆盖当前 `jianli.pdf` 的已知问题：不得把“姓名 沈笑”识别为竞赛经历，不得把“个人技能：”识别为科研经历，不得把“其它经历：”识别为技能。
- 百炼抽取使用 mock，验证 Zod 校验、证据定位、去重、冲突保留与失败降级。
- 数据库测试验证删除材料会级联删除片段和画像事实，同时保留历史面试与复盘。
- 文件测试验证正常删除、数据库失败时恢复隔离文件以及路径越界保护。
- Playwright 覆盖重复上传提示、删除确认、删除后界面刷新、基础解析降级和智能解析重试。
- 手动真实 API 验收使用当前简历，确认项目、竞赛和技能均带有可核对的原文证据；默认自动化测试不产生百炼费用。
